# Hybrid Token: JWT Access + Redis Refresh (by jti)

## Goal
- **Access token**: JWT signed (HS256), stateless, lấy username từ `sub` claim
- **Refresh token**: JWT signed (HS256) như cũ, nhưng lưu `jti` (UUID) vào Redis để validate + revoke
- **User cache**: Redis cache user data, chỉ chọc DB khi cache miss

## Redis Key Design

| Key Pattern | Type | TTL | Value | Purpose |
|---|---|---|---|---|
| `auth:refresh:jti:{jti}` | Hash | 30d | `username`, `created_at` | Refresh token validity check |
| `auth:user:{username}` | Hash | 1h | `id`, `username`, `displayName` | User cache |

Refresh token vẫn là JWT với `jti` = UUID (đã có sẵn trong code hiện tại).

## New Files

1. **`services/RefreshTokenService.java`**
   - `storeRefreshToken(jti, username)` → `HSET auth:refresh:jti:{jti}` + `EXPIRE 30d`
   - `isValidRefreshToken(jti)` → `EXISTS auth:refresh:jti:{jti}`
   - `revokeRefreshToken(jti)` → `DEL auth:refresh:jti:{jti}`
   - `revokeAllUserRefreshTokens(username)` → scan pattern, delete matching

2. **`services/UserCacheService.java`**
   - `getCachedUser(username)` → Redis first, fallback DB + cache
   - `cacheUser(User)` → `HMSET` + `EXPIRE 3600`
   - `invalidateUserCache(username)` → `DEL auth:user:{username}`

3. **`dtos/token/TokenRevokeRequestDto.java`** — Cho logout endpoint

## Modified Files

4. **`services/JwtsService.java`**
   - `tokenObtainPair()`: sau khi generate refresh JWT → extract `jti` → gọi `RefreshTokenService.storeRefreshToken(jti, username)`
   - `refreshToken()`: decode refresh JWT → extract `jti` → `RefreshTokenService.isValidRefreshToken(jti)` → nếu valid thì generate access JWT mới
   - Inject thêm `RefreshTokenService`, giữ nguyên `JwtEncoder`

5. **`services/UserService.java`**
   - `getUserByAuthentication()`: `Jwt` principal → `sub` claim → `userCacheService.getCachedUser(username)` thay vì `repository.findByUsername()`
   - `updateCurrentProfile()`: sau save → `userCacheService.invalidateUserCache(username)`
   - Inject thêm `UserCacheService`

6. **`controllers/UserController.java`**
   - Thêm `POST /api/v1/users/token/revoke/` → decode refresh JWT → extract `jti` → `RefreshTokenService.revokeRefreshToken(jti)`

7. **`resources/application.yml`**
   - Thêm `auth.user-cache-ttl: "PT1H"`

## DB Hit Reduction

| Scenario | Before | After |
|---|---|---|
| Request đầu sau login | 1 DB hit | 1 DB hit (cache miss) |
| Request tiếp theo | 1 DB hit/request | 0 (Redis cache hit) |
| Refresh token | 0 (JWT stateless) | 0 (Redis check by jti) |

## Verification

1. `./gradlew compileJava compileKotlin` — build passes
2. `./gradlew test` — all tests pass
3. Login → access là JWT, refresh là JWT (có jti), Redis có key `auth:refresh:jti:{jti}`
